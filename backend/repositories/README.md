# Repositories

Repositories are the database access layer.

Rules:

- Keep Supabase/Postgres queries here.
- Do not import React code or UI components.
- Do not decide product workflows here; services should do that.
- Keep method names aligned with `backend/interfaces`.

- `feed.repository.js`
- `advocate.repository.js`
- `appointment.repository.js`
- `message.repository.js`
- `ai.repository.js`
